# 参与贡献

感谢你愿意帮助改进 ZJ HTML Studio。

## 提交前

1. 行为变化或较大重构，请先创建 Issue 说明问题和预期结果。
2. 保持改动聚焦，不削弱默认“安全只读”边界。
3. 行为发生变化时，补充或更新对应测试。
4. 在本地运行完整检查：

```bash
npm ci
npm run check
```

## Pull Request 说明

请写清楚：

- 要解决的用户问题
- 采用的实现方式
- 对安全与隐私的影响
- 已执行的验证
- 尚未覆盖的平台或场景

测试夹具、截图和日志中不得包含真实知识库内容、账号凭据、私有链接、本机绝对路径或其他个人数据。
