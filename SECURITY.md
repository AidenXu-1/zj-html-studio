# 安全政策

## 支持范围

安全修复面向 ZJ HTML Studio 当前最新公开版本提供。

## 预览权限边界

- **安全只读**关闭用户页面脚本、常见后台请求、外部子资源和剪贴板。
- **本地交互**允许内联脚本和随机本地会话来源中的脚本，通过 CSP 阻止常见 fetch、XHR、EventSource、WebSocket、外部子资源、Worker、嵌套页、表单、剪贴板和动态求值。
- **可信兼容**可以联网并使用更完整的页面能力，只应对自己生成或已审查页面开启。

三种模式都使用独立随机 `.localhost` 来源和最小本地资源范围。链接或页面导航仍可能离开本地预览。Obsidian 1.12.7 的实际 Chromium 不执行响应头中的 `webrtc 'none'`，DataChannel 与 UDP STUN 探针仍可产生网络活动。因此“本地交互”只承诺阻止已列出的常见请求通道，只适合自己制作并检查过的页面。

本地文件会先以只读句柄打开，再通过该句柄复核真实路径、文件类型与设备/文件身份。中间目录在检查期间被换成符号链接、路径离开授权范围或文件身份发生变化时，请求会停止且不返回文件内容。无法提供稳定文件身份的文件系统会保守拒绝读取。

HTML 正文会在有界内存中形成最多 32 MiB 的不可变快照；UTF-8 校验、Range、搜索桥和最终响应均使用同一份字节。这样即使源文件在响应前被再次保存，也不会把两个版本混合后发送给页面。

安全只读的页内查找会先对 DOM 规模与封闭 Shadow DOM 做保守预检，再使用 Chromium 原生定位。本地交互和可信兼容允许页面脚本改变 DOM，因此只对插件能够安全遍历的可见内容做有界、可取消的分片定位，并在界面标记“部分受限”；插件不会为追求完整结果执行无上限同步扫描。

## 报告安全漏洞

疑似安全漏洞请不要提交公开 Issue。请使用仓库 Security 页面中的 **Report a vulnerability** 私密报告入口：

<https://github.com/AidenXu-1/zj-html-studio/security/advisories/new>

报告时请尽量提供：

- 受影响的插件版本
- 操作系统与 Obsidian 版本
- 可复现步骤
- 可能造成的影响
- 可以安全分享的最小复现文件

请勿上传真实知识库内容、个人资料、访问令牌、私有链接或其他敏感信息。需要提供样例时，请先脱敏并缩减到最小范围。

我们会尽量在七天内确认收到报告。确认问题后，将与报告者协作完成调查、修复和适当披露。
